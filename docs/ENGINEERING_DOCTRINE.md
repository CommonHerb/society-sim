# Engineering Doctrine

## 1. Depth Over Breadth
Add fewer systems with stronger coupling and stronger proofs.

## 2. Explicit Causality
If a behavior cannot be explained, it is not accepted.

## 3. Measured Novelty
Novel design is required, but novelty must survive falsification.

## 4. No Invisible Assumptions
All assumptions about agent knowledge, policy goals, and constraints must be encoded and documented.

## 5. Build for Failure
Design for collapse, corruption, misinformation, and stagnation as first-class states.

## 6. Human-Readable Outputs
Chronicle quality is a core product metric, not cosmetic output.

## 7. Verify Before Celebrate
Every claim requires replay evidence, benchmark evidence, or validation evidence.

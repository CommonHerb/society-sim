# Simulation Program Masterplan

## 0) Mission and Non-Negotiables
- Build a persistent autonomous society simulation that remains meaningful with zero active players.
- Model pressures, incentives, and information distortion; do not script outcomes.
- Preserve uncertainty: no omniscient decision surface for any actor.
- Produce durable historical output (chronicle) that is legible and causally traceable.
- Avoid toy complexity: prefer a small number of deeply coupled systems over many shallow mechanics.

## 1) Product Thesis
This system explores how social order forms, persists, and decays under bounded information, unequal force, and material constraints.  
The primary output is not a score. It is a living historical record generated by endogenous interaction.

## 2) Program Success Criteria
- Technical:
  - Deterministic replay from seed for all simulation ticks.
  - Stable operation through 100,000 ticks with no freeze state.
  - Horizontal scalability to 100k settlements in batch simulation.
- Simulation:
  - Emergent diversity across seeds (no single attractor regime).
  - No inevitable war/collapse; only shifting probabilities under pressure.
  - Detectable regime transitions (expansion, consolidation, fragmentation, stagnation).
- Narrative:
  - Chronicle entries are coherent and causally linked.
  - Distortion layer produces conflicting but plausible accounts.
- Operational:
  - Full observability, incident runbooks, snapshot/restore, and versioned tuning.

## 3) Architectural Decisions (ADR-Level)
- `ADR-001`: Deterministic simulation core with explicit RNG stream partitioning.
- `ADR-002`: Event-sourced history (delta log) plus periodic snapshots.
- `ADR-003`: Ground truth and reported truth are separate first-class models.
- `ADR-004`: Institutions are policy engines with bounded rationality, not perfect optimizers.
- `ADR-005`: Legitimacy is inferred from belief topology, not directly assigned.
- `ADR-006`: All balancing parameters are externalized and versioned.

## 4) Domain Model v1 (Canonical)
- Settlement:
  - `id`, `region_id`, `faction_control`, `population`, `food_stock`, `goods_stock`
  - `security_capacity`, `extraction_rate`, `trade_connectivity`, `border_exposure`
  - `institution_profiles`, `infrastructure_index`, `environmental_stress`
- Institution:
  - `type` (`military`, `market`, `council`, `press`, `clergy`, `judicial`)
  - `capacity`, `credibility`, `capture_risk`, `agenda_vector`, `resource_access`
- Agent Aggregate (not full micro-agent at v1):
  - `group_id`, `role_mix`, `trust_weights`, `grievance_index`, `compliance_index`
- Belief Graph:
  - Nodes: institutions + population blocs
  - Edges: weighted trust and communication throughput
  - Per-edge distortion/latency characteristics
- Event:
  - `event_type`, `trigger_factors`, `affected_entities`, `severity`, `timestamp`
  - `ground_truth_payload`, `reported_variants[]`

## 5) Tick Pipeline (Authoritative Order)
1. Load current state + parameter version.
2. Update material pressures (food/goods/security/trade/strain).
3. Update institutional capacities and constraints.
4. Evaluate event hazard functions and sample weighted outcomes.
5. Apply event consequences to ground truth.
6. Generate reported truth variants per source (distortion, suppression, lag).
7. Propagate belief updates on trust graph.
8. Recompute legitimacy from belief distribution and institutional credibility.
9. Evaluate expansion/decay transitions.
10. Emit chronicle artifacts and telemetry.
11. Persist delta + optional snapshot.

## 10) Risk Register (Top 10)
1. Overfitting to one dynamic regime.
   - Mitigation: multi-seed sensitivity sweeps; reject single-attractor parameter sets.
2. False complexity (many variables, weak coupling).
   - Mitigation: enforce coupling audits and causal contribution tests.
3. Narrative incoherence from event spam.
   - Mitigation: editorial pacing controls and event salience thresholds.
4. Hidden non-determinism across environments.
   - Mitigation: deterministic build matrix and replay CI gate.
5. Governance capture becomes irreversible.
   - Mitigation: structural decay mechanics and institutional turnover pressures.
6. Trade/economy dominates all outcomes.
   - Mitigation: balanced influence constraints and capped elasticity.
7. Frontier lock-in due to dead map zones.
   - Mitigation: reversion/recolonization pathways.
8. Operator tuning causes accidental regime collapse.
   - Mitigation: staged rollout, canary runs, rollback in minutes.
9. Computational cost scales superlinearly.
   - Mitigation: adjacency pruning, sparse graph operations, profiling gates.
10. Confusing outputs undermine trust in simulation.
   - Mitigation: causal traceability for every visible artifact.
